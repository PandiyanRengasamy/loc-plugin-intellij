# 🧪 Testing LOC Plugin with GitHub Copilot Chat

## Overview

This guide explains how to test the **GenAI LOC Tracker** plugin while using GitHub Copilot Chat in IntelliJ IDEA to verify that the plugin correctly tracks lines of code generated by Copilot.

---

## ✅ Prerequisites

Before testing, ensure:

1. ✅ IntelliJ IDEA 2025.1 or later installed
2. ✅ Java 21 or later installed
3. ✅ GitHub Copilot plugin installed and enabled
4. ✅ LOC Plugin built and ready to test
5. ✅ Backend API running (or ready to test CSV fallback)

---

## 🎯 Test Scenarios

### Scenario 1: Basic Copilot Code Generation

**Objective**: Verify plugin tracks code when Copilot generates content

**Steps**:

1. **Open IntelliJ IDEA**
   - Create a new Java project or open existing one
   - Create a new Java file (e.g., `TestClass.java`)

2. **Enable Copilot Chat**
   - Open Copilot Chat (View → Tool Windows → GitHub Copilot Chat)
   - Or use keyboard shortcut (typically Ctrl+Shift+A on Windows)

3. **Request Code Generation**
   - In Copilot Chat, type:
     ```
     Write a Java method to calculate factorial of a number with proper error handling
     ```

4. **Accept Copilot Suggestion**
   - Copy/paste the generated code into your file
   - Or click "Accept" if inline suggestion appears

5. **Verify Plugin Tracking**
   - Check IntelliJ status bar for LOC plugin status
   - Monitor: Settings → Tools → GenAI LOC Tracker
   - Verify event was recorded in logs

**Expected Result**:
```
✅ Plugin detects Copilot-generated code
✅ Lines added count is incremented
✅ Confidence score recorded (~0.85-0.95)
✅ Event timestamp captured
✅ Event sent to backend (or saved to CSV if offline)
```

### Method 1: Console Logs

**View IntelliJ Logs**:
1. Help → Show Log in Explorer
2. Search for "LOC" or "GenAiLoc"
3. Look for debug messages

**Expected Logs**:
```

[DEBUG] CodeEventRequest created: file=Test.java dev=<id> tool=GitHub Copilot +lines=10 ~lines=2 -lines=1 genAi=true confidence=0.92 session=<uuid>
[INFO] EventDispatcher: disposing — final stats: Sent=5 Failed=0 Pending=0
```

### Method 2: CSV Fallback File

**Check Offline Storage**:
```bash
# Windows
type %USERPROFILE%\.genai-loc\fallback-20260402.csv

# Linux/Mac
cat ~/.genai-loc/fallback-20260402.csv
```

**Expected Format**:
```
developerId,developerName,projectId,sprintId,filePath,fileName,ideType,genAiTool,developmentMode,linesAdded,linesModified,linesDeleted,genAiGenerated,genAiConfidenceScore,eventTimestamp,sessionId
dev-123,John Doe,PROJ-001,SPRINT-1,/path/to/Test.java,Test.java,IntelliJ IDEA,GitHub Copilot,Production,10,2,1,true,0.92,2026-04-02T10:30:00Z,session-abc123
```

### Method 3: Status Bar Indicator

**Monitor Plugin Status**:
1. Look at bottom right corner of IDE
2. Check for plugin status indicator
3. Hover to see statistics: `Sent=X Failed=Y Pending=Z`

### Method 4: Backend Verification

**If Backend is Running**:
1. Check backend API logs
2. Verify POST requests received
3. Confirm request structure and data

**Test Endpoint**:
```bash
# Check what requests were received
curl http://localhost:8080/api/events

# Or view in backend logs
tail -f backend.log | grep "LOC\|event"
```

## 🐛 Troubleshooting Test Issues

### Issue: Plugin Not Tracking Events

**Diagnosis**:
1. Check if plugin is enabled (Settings → Tools → GenAI LOC Tracker)
2. Verify Copilot is actually generating code
3. Check logs for errors

**Solution**:
```
1. Restart IDE
2. Verify plugin.xml is loaded
3. Check for exceptions in logs
4. Rebuild plugin if needed
```

### Issue: CSV File Not Created

**Diagnosis**:
1. Backend might be reachable
2. Directory might not have write permissions
3. Events might have been sent successfully

**Solution**:
```bash
# Check directory exists and permissions
ls -la ~/.genai-loc/

# Or Windows
dir %USERPROFILE%\.genai-loc\

# Grant write permissions if needed
chmod 755 ~/.genai-loc/
```

### Issue: Wrong Confidence Scores

**Diagnosis**:
1. Confidence detection might be off
2. Code pattern recognition might be wrong
3. GenAI detection logic might have issues

**Solution**:
1. Check GenAiToolDetector.java implementation
2. Review confidence scoring logic
3. Verify Copilot markers in code

### Issue: Events Not Sent to Backend

**Diagnosis**:
1. Backend URL might be wrong
2. Network connectivity issue
3. Backend service down

**Solution**:
```bash
# Test backend connectivity
curl -X POST http://localhost:8080/api/events \
  -H "Content-Type: application/json" \
  -d '{"test": "data"}'

# Check if service is running
netstat -an | grep 8080
```
